# GIS 基础概念

## 1. 坐标系 & 投影

### 1.1 地理坐标系（Geographic CRS）

- **WGS‑84**（EPSG:4326）是全球通用的经纬度坐标系，单位为 **度**（°）。
- 典型表示方式：`[longitude, latitude]`，例如北京的经纬度约为 `[116.38, 39.90]`。
- 大多数 GIS 数据（GeoJSON、KML）默认采用此坐标系。

### 1.2 投影坐标系（Projected CRS）

- 为了在平面上绘制地图，需要把球面坐标映射到平面，这就是投影。
- **[Web Mercator](../../wiki/web-mercator.md)**（EPSG:3857）是 Web 地图（OpenStreetMap、Google Maps、Mapbox）常用的投影，单位为 **米**。
- 其他常见投影：
  - **UTM**（Universal Transverse Mercator），分区使用 EPSG:326xx（北半球）/ EPSG:327xx（南半球）。
  - **Lambert Conformal Conic**，适用于中纬度国家/地区。

### 1.3 投影转换库

- **proj4js**：JS 实现的 PROJ.4，支持上百种 EPSG 投影。
  
  proj4js 的作用可以理解为：
  
  > **在不同坐标系之间做数学转换**
  
  本质就是：
  
  ```javascript
  proj4(源坐标系, 目标坐标系, 坐标)
  ```
  
  示例：
  
  ```js
  // 将 WGS84 坐标转换为 Web Mercator
  const proj4 = require('proj4');
  const wgs84 = 'EPSG:4326';
  const webMerc = 'EPSG:3857';
  const [x, y] = proj4(wgs84, webMerc, [120, 30]); // → [13358338.89, 3503549.84]
  ```
  
- **@turf/projection**：基于 Turf.js 的投影工具，常与空间分析函数一起使用。

## 2. GeoJSON 基础

- GeoJSON 是一种基于 JSON 的 **地理要素** 描述格式，广泛用于前端地图库（Leaflet、OpenLayers、Mapbox GL）。
- 常见几类要素：`Point`、`LineString`、`Polygon`、`MultiPoint`、`MultiLineString`、`MultiPolygon`、`GeometryCollection`。
- **结构示例**（点要素集合）：
  ```json
  {
    "type": "FeatureCollection",
    "features": [
      {
        "type": "Feature",
        "geometry": { "type": "Point", "coordinates": [120.0, 30.0] },
        "properties": { "name": "示例点" }
      }
    ]
  }
  ```
- 在前端加载方式（以 Leaflet 为例）：
  ```js
  fetch('data/sample.geojson')
    .then(r => r.json())
    .then(geojson =>
      L.geoJSON(geojson, {
        pointToLayer: (feat, latlng) => L.marker(latlng),
        onEachFeature: (feat, layer) => {
          if (feat.properties && feat.properties.name) {
            layer.bindPopup(feat.properties.name);
          }
        },
      }).addTo(map),
    );
  ```

## 3. 常用空间分析库

- **turf.js**：提供缓冲区、相交、测距、聚合等 GIS 分析功能，完全基于 GeoJSON。
- **@turf/projection**：与 proj4js 配合，完成坐标投影转换后再进行分析。
